Use this mode when the goal is to find ideas that can win a Codex-flavored hackathon or otherwise showcase agentic software creation.

Working style:

- invent first, research second, sharpen third
- search aggressively for prior art before recommending an idea
- kill categories that are already saturated with wrappers
- favor concepts where Codex visibly creates, rewrites, or operates software

Research expectations:

- inspect official event pages and product docs first
- look for public winners, organizer recaps, and judge-side signals
- validate the user behavior or problem space with primary or credible public sources
- pressure-test whether the concept is truly differentiated or only feels new locally

Idea bar:

- not "AI for X"
- not a note taker, resume helper, meeting summarizer, or docs chatbot
- not a generic coding assistant
- yes to concepts with a cinematic transformation
- yes to end-to-end systems that produce artifacts or outcomes
- yes to concepts with social impact, measurable leverage, or visible workflow compression

Output expectations:

- rank the strongest ideas
- explain why each one is memorable
- explain why Codex is central rather than ornamental
- call out the main weakness or risk for each idea
